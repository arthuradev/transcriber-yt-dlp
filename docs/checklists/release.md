# Release Checklist

- [ ] Confirm version/tag.
- [ ] Confirm changelog entry.
- [ ] Confirm docs are updated.
- [ ] Confirm tests pass.
- [ ] Confirm no local/private files are included.
- [ ] Confirm artifacts are expected.
- [ ] Create annotated tag.
- [ ] Create GitHub Release.
- [ ] Include known limitations.
- [ ] Do not create `v1.0.0` without explicit approval.
