# Architecture Decision Records

Use ADRs for decisions that affect architecture, safety, tooling, packaging, data handling, or long-term project direction.

Preferred template:

```md
# ADR — Title

## Status
Proposed | Accepted | Deprecated | Superseded

## Context
What problem or constraint led to this decision?

## Decision
What did we decide?

## Consequences
What becomes easier, harder, or riskier?

## Notes for AI agents
What must future agents preserve?
```
