# Project Principles

## Product principles
- Beautiful but not fragile.
- Safe by default.
- Configurable by user.
- Windows-first and Windows-only for now.
- No secrets in public repo.
- No hidden destructive behavior.
- Clear progress and clear results.

## Engineering principles
- Prefer boring, maintainable code.
- Prefer explicit contracts over clever coupling.
- Prefer small changes over large rewrites.
- Prefer tests around rules, boundaries, and safety.
- Prefer clean errors over silent failure.
- Prefer local configuration over hardcoded personal settings.
- Prefer event-based progress over parsing fragile output.

## AI development principles
- AI agents must read the project docs.
- AI agents must keep changes scoped.
- AI agents must run quality gates.
- AI agents must update docs and changelog.
- AI agents must not invent architecture.
- AI agents must stop at phase completion.
